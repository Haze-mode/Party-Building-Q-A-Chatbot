# 党建知识问答系统

> 基于 GLM-4 大模型的智能党建知识问答平台，融合**模型微调**与**文档检索增强**双重技术

## 项目简介

本项目是一个面向党建学习的智能问答系统，结合了**大语言模型微调**和**文档知识库检索**两项核心技术，为党员和群众提供准确、权威、即时的党建知识问答服务。

### 核心技术架构

```
用户提问
  ↓
┌─────────────────────────────────────┐
│  双重知识增强机制                    │
│                                     │
│  ┌─────────────┐  ┌──────────────┐ │
│  │  模型微调    │  │  文档知识库   │ │
│  │  (Fine-tune)│  │  (KB)        │ │
│  │             │  │              │ │
│  │ 学习党建    │  │ 检索最新     │ │
│  │ 专业术语    │  │ 政策文件     │ │
│  │ 回答风格    │  │ 实时更新     │ │
│  │ 领域知识    │  │ 文档内容     │ │
│  └─────────────┘  └──────────────┘ │
│         ↓              ↓            │
│  ┌─────────────────────────────┐   │
│  │    融合生成准确回答          │   │
│  └─────────────────────────────┘   │
└─────────────────────────────────────┘
  ↓
返回答案
```

### 技术亮点

#### 1. 模型微调（Fine-tuning）
- 使用 GLM-4 基础模型进行**领域适应性微调**
- 训练数据涵盖党建理论、政策解读、党史知识等
- 使模型掌握党建领域的专业术语和回答风格
- 提升模型对党建问题的理解和回答质量

#### 2. 知识库检索增强
- 构建文档知识库，存储最新政策文件、学习资料
- 支持 PDF、Word、TXT 等多种格式文件上传
- 通过文档检索匹配相关片段，辅助模型生成回答
- 知识库可随时更新，确保回答时效性

#### 3. 双重机制优势

| 能力 | 模型微调 | 知识库检索 |
|------|----------|------------|
| **专业术语** | ✅ 掌握党建术语 | ❌ |
| **回答风格** | ✅ 符合党建语境 | ❌ |
| **最新政策** | ❌ 需要重新训练 | ✅ 实时更新 |
| **具体文件** | ❌ 记忆有限 | ✅ 精准检索 |
| **事实准确性** | 可能幻觉 | ✅ 基于原文 |

**两者结合 = 专业 + 准确 + 时效**

## 功能特性

### 🎯 核心功能

- **智能问答** - 基于 GLM-4 模型的党建知识问答
- **会话管理** - 支持多轮对话，上下文理解
- **历史记录** - 查看当前会话的所有对话
- **导出对话** - 一键导出对话记录为 DOC 文档

### 🔐 用户系统

- **游客模式** - 无需登录即可体验基础问答
- **用户登录** - 用户名/密码认证，Token 鉴权
- **角色权限** - 普通用户 vs 管理员权限分离

### 📚 知识库管理（管理员专属）

- **文件上传** - 支持拖拽/点击上传，支持 PDF、Word、TXT 等格式
- **文件列表** - 查看已上传的所有知识库文件
- **文件删除** - 删除不需要的知识库文件
- **重载知识库** - 重新索引知识库，让 AI 检索最新内容

## 快速开始

### 1. 环境要求

- 后端服务运行在 `http://localhost:6006`
- 需要启动 GLM-4 微调模型服务
- 需要配置向量知识库服务

### 2. 配置接口地址

编辑 `chatJs/auth.js` 文件，修改 API 地址：

```javascript
const API_BASE_URL = 'http://localhost:6006';  // 修改为你的后端地址
```

### 3. 启动前端

直接双击 `index.html` 即可在浏览器中打开（建议使用 Chrome 浏览器）

> **提示**：由于浏览器安全限制，建议使用本地服务器运行：
> ```bash
> # 使用 Python 启动本地服务器
> python -m http.server 8080
> # 然后访问 http://localhost:8080
> ```

### 4. 登录账号

- **管理员账号**：`admin` / `admin123`
- **游客模式**：点击「以游客身份体验」

## 后端接口说明

### 认证接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/login` | POST | 用户登录 |
| `/api/logout` | POST | 用户登出 |

### 聊天接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/chatbot` | POST | 发送问题，获取回答 |
| `/api/session/clear` | GET | 清空当前会话 |

### 知识库接口（管理员）

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/kb/files` | GET | 获取知识库文件列表 |
| `/api/kb/upload` | POST | 上传文件到知识库 |
| `/api/kb/files/<filename>` | DELETE | 删除知识库文件 |
| `/api/kb/reload` | POST | 重载知识库索引 |
