# Party-Building-Q-A-Chatbot
🚀 技术特色: • RAG 架构: 检索增强生成,减少模型幻觉 • 双引擎检索: TF-IDF + FAISS,兼顾精度与效率   • LoRA 微调: 仅 17M 参数量,显存占用低至 22GB • 三级缓存: 问答/检索/会话分层缓存,命中率优化 • 异步服务: Tornado 框架,支持高并发访问 • 模块化设计: 清晰的代码结构,易于扩展维护  📊 性能指标: • 模型: GLM-4-9B-Chat (90亿参数) • 微调方法: LoRA (rank=16, alpha=32) • 检索 Top-K: 3 • 缓存容量: 问答500条/检索1000条/会话2000条 • 响应延迟: &lt; 3秒 (含检索+生成)
